### 格式错误与修正 ⚠️

#### 1. EIA 格式 (反复犯错)
**❌ 错误**: 用【库存】【供应】【进出口】结构化模板（那是周三定时 EIA 专项报告格式）
**✅ 正确**: 紧凑叙述段落
```
EIA周报（截至X月X日）显示，美国除却战略储备的商业原油库存增加XXX万桶至X亿桶；汽油库存...；原油产量...；炼厂加工量...。
```

#### 2. 成品油格式 (反复犯错)
**❌ 错误**: `"石脑油vs迪拜裂解利润从-3.02回升至+1.83"` / `"裂解利润+72.29美元/桶（-4.75）"`
**✅ 正确**: `"石脑油加工利润上涨4.85美元至1.83美元/桶（CFR日本 vs 迪拜）"`
- 格式：`[品种]利润上涨/下跌X美元至X美元/桶` (变化量 + 绝对值)
- 结尾必须：`"整体来看，迪拜综合裂解利润下跌/上涨X美元至X美元/桶。"`

#### 3. TD3C 第一句格式 (反复犯错，"已经错过很多很多次" - Tianshu原话)
**❌ 错误**: `"本周TD3C波罗的海评估收于W413.89..."` / `"WS413.89，TCE约..."`
**✅ 正确**: `"TD3C VLCC 运费本周[大幅上扬/小幅回落/趋于平稳]，收于 W[数字]，约[X]万美元/船，折合[X]美元/桶。"`
- WS → 万美元: WS/100 × 平价($10.72/MT) × 载货量(265,000MT) ÷ 10,000
- 美元/桶: 万美元×10,000 ÷ 约194万桶

## 跨渠道消息限制

### 问题
- 从 Telegram context 启动的任务无法发送到 WhatsApp
- 子任务 (subagent) 也继承父任务的 channel binding
- 错误: `"Cross-context messaging denied: action=send target provider \"whatsapp\" while bound to \"telegram\"."`

### 解决方式
- 直接在 Telegram topic 发送文件
- 或从 WhatsApp 发起任务

## 教训总结

1. **周报格式规则必须严格遵守**，Tianshu 对格式要求极其精确，反复犯同样错误会失去信任
2. **EIA/成品油/TD3C 三个格式是最容易犯错的地方**，已加固到 SKILL.md
3. **市场因素排序逻辑**: 产业(供需) → 机构观点 → 宏观，不能跳跃
4. **删除冗余内容**: 价格表已有的数据不在正文重复，裂解利润细节过于技术性
