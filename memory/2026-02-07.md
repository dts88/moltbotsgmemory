## Platts API 结构分析
Tianshu 问 Platts 有哪些接口和分类，我做了完整梳理：

### News Insights API
- `/news-insights/v1/search/heards` - 市场听闻
- `/news-insights/v1/search/story` - 新闻文章
- `/news-insights/v1/content/{id}` - 内容详情
- **不支持直接分类过滤**，需本地用关键词筛选

### 内容分类字段 (envelope.properties)
- `commodity`: 商品类别
- `sector`: 服务板块
- `geography`: 地理区域
- `subjectArea`: 主题领域
- `alertLine`: 服务线/页码

### Market Data API
- `/market-data/v3/value/current/symbol` - 当前价格
- `/market-data/v3/value/history/symbol` - 历史价格
- Filter: `symbol:"CODE"`, `assessDate>="YYYY-MM-DD"`

### Reference Data API
- `/market-data/reference-data/v3/search` - 代码元数据
- 包含 commodity, mdc, delivery_region 等

### 缺失功能
- ❌ Tender 招标 API
- ❌ MOC 窗口实时数据
- ❌ 按商品/地区直接过滤搜索
